SLIDE 1 — Title

"Hi, I’m Arda. Today I’m going to present Calen — an AI-powered calendar you can control through conversation.

I built this to explore removing the need to download apps and letting users interact through messaging."

SLIDE 2 — The Problem

"Here’s the problem.

Today, managing your calendar is manual. You go through forms, pick dates, and check for conflicts. Even small changes take multiple steps.

And even if you want to use AI, it’s not built in. Apps like Google Calendar don’t have a native AI assistant. You need a separate app like Google Gemini.

So you end up switching between apps and doing things manually.

More importantly — everything still depends on downloading an app."

SLIDE 3 — The Solution

"So the idea behind Calen is simple — make the calendar fully conversational and accessible through messaging.

Instead of forcing users to download an app, they can just send a message.

Calen works in multiple ways:

A mobile app
Messaging channels (e.g. iMessage)
And both use the same AI system

The key idea is: the interface is the conversation — not the app."

SLIDE 4 — Features (keep simple, but highlight messaging)

"Here’s what it can do.

You can create events in one message — even bulk ones.

It detects conflicts and suggests better times automatically.

You can update or delete events by just typing.

You can ask about your schedule like ‘what do I have this week?’

It also supports recurring events, finds events online, sends reminders, and organizes your schedule.

All of this works the same way in the app or over messaging."

SLIDE 5 — Agent Architecture

"Here’s how it works.

Every message goes to a Router Agent that understands what the user wants.

If it’s a calendar task, it goes to the Scheduling Agent.
If it’s something like finding events, it goes to a search agent.

Then the system:

Extracts details
Checks for conflicts
Suggests better times
Executes the action

Finally, notifications are sent if needed."

SLIDE 6 — Tools & Services

"The agents use tools for calendar actions like create, update, delete, and conflict checking.

In the background:

Notifications are sent before events
Redis stores conversation memory
Webhooks are verified and handled securely"
SLIDE 7 — Tech Stack

"Quick overview of the stack.

Backend: Python with FastAPI and LangGraph
AI: OpenAI
Database: PostgreSQL and Redis

Mobile: React Native with Expo

Integrations:

Tavily for web search
Expo for notifications"
SLIDE 8 — Messaging integration

"This is where messaging fits in.

The user sends a message → a messaging bridge forwards it → our backend processes it → and sends a response back.

There’s no app required for users who prefer messaging.

Users can manage their calendar through conversation.

Accounts can be created automatically, and users can optionally link to the mobile app.

This shows how messaging can act as the main interface, not just a notification layer."
